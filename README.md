# Claude Code Toolkit

개인용 Claude Code 스킬 모음입니다. 반복적으로 사용하는 작업 패턴을 스킬과 스크립트로 구현하여 Jira 티켓 관리, Trello 세션 추적, 커밋 메시지 생성 등 개발 워크플로우를 자동화합니다.

> **AI 워크플로우 자동화는 공통화보다 개인화가 중요하다고 생각합니다.**
>
> 각자의 작업 방식과 맥락에 맞게 커스터마이징하여 사용하시면 좋겠습니다.

## Tips

### 1. Alias로 워크플로우 감싸기

`claude` 명령어를 직접 실행하는 대신, 자주 사용하는 패턴을 alias로 만들어 사용하고 있습니다.

저는 터미널 세션을 최소 5개 이상 끄지 않고 켜두는 편인데, 시간이 지나면 어디서 무슨 작업을 했는지 헷갈리기 시작합니다. 게다가 모든 터미널이 같은 프로젝트가 아니라 여러 프로젝트가 섞여 있어서 더 혼란스럽습니다. 그래서 Trello로 각 세션의 히스토리와 컨텍스트를 추적하고 있습니다. Jira가 실제 티켓 일감을 관리한다면, Trello는 Claude 세션 하나(터미널 하나)를 관리하는 용도입니다.

`cs`라는 alias를 만들어 Claude Code와 Trello를 연동하고 있습니다. `cs`를 실행하면 Claude Code가 시작되기 전에 이전 세션의 Trello 카드를 자동으로 연결하고, 세션이 종료될 때 카드 상태를 어떻게 할지 물어봅니다. 일시정지, 완료, 긴급 중에서 선택하면 Trello 카드가 자동으로 이동합니다.

```bash
alias cs="~/.claude-trello/claude-session"
```

이렇게 하면 작업 시작과 종료 시점에 수동으로 Trello를 관리하지 않아도 됩니다. Claude Code를 감싸는 래퍼 스크립트를 만들고, 자신만의 전처리/후처리 로직을 추가하는 방식을 추천합니다.

### 2. 결정적 로직과 비결정적 로직 분리하기

스킬이나 서브에이전트를 만들 때, LLM이 꼭 필요한 작업과 그렇지 않은 작업을 구분하는 것이 좋다고 생각합니다.

API 호출, 브랜치명 파싱, 상태 변경처럼 정해진 규칙대로 동작하는 것은 스크립트로 처리하고, 코멘트 작성, 내용 요약, 명령어 해석처럼 맥락을 이해하고 판단이 필요한 것은 LLM에게 맡기는 방식입니다.

이렇게 분리하면 LLM 호출 횟수를 줄여 비용과 지연을 낮출 수 있고, 결정적인 부분은 항상 동일한 결과를 보장할 수 있습니다. 이 저장소의 Jira 스킬도 TypeScript 스크립트가 API 호출과 데이터 변환을 담당하고, LLM은 사용자 명령어 해석과 출력 포맷팅만 처리하도록 구성했습니다.

### 3. 큰 작업은 설계 먼저, 작은 작업은 대화로 시작

프로젝트 규모에 따라 접근 방식을 다르게 하고 있습니다.

새로운 기능 개발이나 아키텍처 변경처럼 큰 주제는 AI-DLC 같은 방법론을 활용해 상세한 설계 문서를 먼저 작성합니다. 목표, 제약 조건, 단계별 계획을 명확히 정리한 다음 구현을 시작하면 방향을 잃지 않는다고 느꼈습니다.

반면 버그 수정이나 간단한 개선처럼 작은 작업은 일단 프롬프트로 배경을 설명하고 대화를 시작합니다. AI가 맥락을 이해하면 세세하게 작업을 지시하면서 진행하고, 이 과정에서 반복되는 패턴이 보이면 그때 자동화를 고민하는 편입니다.

### 4. 일단 시작해보기

완벽한 프롬프트나 완벽한 스킬을 만들려고 하지 않습니다.

아이디어가 떠오르면 일단 시작해봅니다. 부족한 부분은 사용하면서 발견되고, 그때 개선하면 된다고 생각합니다. 이 저장소의 Jira, Trello 스킬도 처음에는 단순한 스크립트 하나로 시작했습니다. 사용하면서 불편한 점을 하나씩 고치다 보니 지금의 형태가 되었습니다.

이런 방식으로 시작해서 별도 프로젝트로 발전한 것들도 있습니다. [genai-commit](https://github.com/Seungwoo321/genai-commit)은 커밋 메시지 생성 스크립트에서 시작했고, [genai-sonar-lint](https://github.com/Seungwoo321/genai-sonar-lint)는 ESLint/SonarJS 분석 자동화에서 출발했습니다. [Wireweave](https://wireweave.ai)는 와이어프레임 DSL을 만들어보자는 아이디어에서 시작했습니다. 그 외에도 여러 사이드 프로젝트를 진행 중입니다.
