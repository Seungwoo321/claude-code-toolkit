# AI-DLC 셋팅 프롬프트 2: AI-DLC 적용방식 보고서

## 개요

이 프롬프트는 프로젝트 목적/목표 보고서를 기반으로 프로젝트 특성을 분석하고, 적합한 설계 방법론과 아키텍처 패턴을 선정하는 보고서를 생성합니다. 이 보고서는 코드 예시 없이 의사결정의 전문적 근거를 포함합니다.

## 참조 백서

- 원본 AI-DLC 백서 (`.claude/docs/ai-dlc-whitepaper-ko.md`)
- 확장 AI-DLC 백서 (`.claude/docs/ai-dlc-extended-whitepaper.md`)

---

## 태그 패턴 (범용)

```
당신의 과제: 프로젝트 목적/목표 보고서를 기반으로 AI-DLC 적용방식 보고서를 작성합니다.

먼저 다음 백서를 읽고 AI-DLC 방법론을 이해합니다.

1. 원본 AI-DLC 백서 (`.claude/docs/ai-dlc-whitepaper-ko.md`)를 읽고 AI-DLC의 핵심 철학과 10가지 원칙을 이해합니다.

2. 확장 AI-DLC 백서 (`.claude/docs/ai-dlc-extended-whitepaper.md`)를 읽고 프로젝트 분석 기반 적용 방식을 이해합니다. AI-DLC는 특정 방법론을 강제하지 않으며, 프로젝트 특성에 맞는 접근방식을 선택해야 함을 파악합니다.

백서 이해가 완료되면, `docs/aidlc-docs_{주제}/project-goal-report.md` 파일을 읽고 프로젝트의 목적, 목표, 핵심 기능, 기술 환경, 제약사항을 파악합니다.

프로젝트 특성을 분석합니다. 프로젝트 유형, 복잡도, 통합 요구사항, 품질 속성 우선순위 등을 평가합니다. 분석 결과를 정리하여 제시하고, [Question]과 [Answer] 태그를 사용하여 분석 결과가 정확한지 검증을 요청합니다.

분석 결과가 검증되면, 프로젝트에 적합한 설계 방법론과 아키텍처 패턴을 비교 분석합니다. 각 후보의 장단점을 프로젝트 맥락에서 평가하고, 최종 선정과 그 근거를 정리합니다. AI-DLC 각 단계(Inception, Construction, Operations)에서 선정된 방법론과 아키텍처를 어떻게 적용할 것인지 계획을 수립합니다.

`docs/aidlc-docs_{주제}/application-approach-report.md` 파일에 보고서를 작성합니다. 보고서는 산문형으로 자연스럽게 서술합니다.

보고서 작성이 완료되면 최종 검토를 요청합니다. 수정이 필요하면 피드백을, 승인하면 다음 단계(폴더 생성 및 프롬프트 세트 생성)로 진행함을 안내합니다.
```
