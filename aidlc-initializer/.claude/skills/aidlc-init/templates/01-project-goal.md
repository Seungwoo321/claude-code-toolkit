# AI-DLC 셋팅 프롬프트 1: 프로젝트 목적/목표 보고서

## 개요

이 프롬프트는 사용자로부터 프로젝트 정보를 수집하고, 프로젝트의 목적과 목표를 정리한 보고서를 생성합니다.

## 참조 백서

- 원본 AI-DLC 백서 (`.claude/docs/ai-dlc-whitepaper-ko.md`)
- 확장 AI-DLC 백서 (`.claude/docs/ai-dlc-extended-whitepaper.md`)

---

## 태그 패턴 (범용)

```
당신의 과제: AI-DLC 방법론을 적용하기 위해 프로젝트 정보를 수집하고, 프로젝트 목적/목표 보고서를 작성합니다.

먼저 다음 백서를 읽고 AI-DLC 방법론을 이해합니다.

1. 원본 AI-DLC 백서 (`.claude/docs/ai-dlc-whitepaper-ko.md`)를 읽고 AI-DLC의 핵심 철학을 이해합니다. 대화 방향의 역전, 볼트 사이클, 계획-승인-실행 패턴이 무엇인지 파악합니다.

2. 확장 AI-DLC 백서 (`.claude/docs/ai-dlc-extended-whitepaper.md`)를 읽고 프로젝트 분석 기반 적용 방식을 이해합니다. AI-DLC는 특정 방법론을 강제하지 않으며, 프로젝트 특성에 맞는 접근방식을 선택해야 함을 파악합니다.

백서 이해가 완료되면, [Question]과 [Answer] 태그를 사용하여 프로젝트에 대한 기본 정보를 수집합니다. 작업 주제, 프로젝트 상태(신규/리팩토링/기능추가), 프로젝트 설명, 그리고 생성될 프롬프트의 패턴(태그 패턴 또는 Claude Code 패턴)을 질문합니다.

정보 수집이 완료되면 분석을 수행합니다. 기존 프로젝트인 경우 의존성 파일, 설정 파일, 디렉토리 구조 등을 분석합니다. 분석이 완료되면 결과를 정리하여 제시하고, 분석 결과가 정확한지 검증을 요청합니다.

분석 결과가 검증되면 `docs/aidlc-docs_{주제}/project-goal-report.md` 파일에 보고서를 작성합니다. 보고서는 산문형으로 자연스럽게 서술합니다.

보고서 작성이 완료되면 최종 검토를 요청합니다. 수정이 필요하면 피드백을, 승인하면 다음 단계(AI-DLC 적용방식 보고서 작성)로 진행함을 안내합니다.
```
