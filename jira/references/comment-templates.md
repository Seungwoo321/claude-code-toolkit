# 코멘트 템플릿

작업 유형별 권장 코멘트 구조입니다.

## 기능 개선 / 버그 수정

```markdown
## 작업 내용

(한 줄 요약)

### 배경
- 기존: (이전 상태)
- 문제: (발생한 문제)
- 해결: (해결 방법)

### 결과
- (개선된 점 1)
- (개선된 점 2)
```

## 리팩토링

```markdown
## 작업 내용

(한 줄 요약)

### 주요 변경 사항
- (변경 1)
- (변경 2)

### 결과
- (개선 효과)
```

## 조사/분석

```markdown
## 분석 결과

### 현황
- (현재 상태 요약)

### 발견 사항
- (발견 1)
- (발견 2)

### 권장 조치
- (조치 1)
- (조치 2)
```

## 코멘트 작성 규칙

- 코드, 파일명, 경로는 백틱으로 감싸기: \`Button.tsx\`
- 마크다운 링크 형식 피하기
- 코드 예시나 수정 파일 목록은 포함하지 않기
- 간결하고 명확하게 요약
